# External Integrations

**Analysis Date:** 2026-04-14

## APIs & External Services

### Discord API
- **Purpose:** Bot commands, voice channel connectivity, message/embed delivery, user interaction handling
- **SDK/Client:** `discord.py>=2.7.0`
- **Auth:** Bot token from `config.yaml` (`bot_token`)
- **Key Files:** 
  - `main.py` - Bot initialization and event handlers
  - `commands.py` - Command definitions and interactions (buttons, voice)
- **Features Used:**
  - Voice state management (joining/leaving channels)
  - Text channels (command input, status messages)
  - Intents: `message_content`, `voice_states`, `guilds`, `guild_reactions`
  - Embeds with thumbnails and clickable links
  - Discord Button interactions (vote buttons, player controls)
  - Message context (requester tracking)
- **Rate Limits:** Standard Discord API rate limits (no custom mitigation visible)
- **Webhooks:** None (bot is event-driven, not webhook-based)

### YouTube (via yt-dlp)
- **Purpose:** Audio stream extraction, metadata retrieval, format selection, signature solving
- **SDK/Client:** `yt-dlp[default]` + `yt-dlp-ejs[all]` + custom bgutil plugin
- **Auth:** 
  - No API key required
  - PO (Proof of Origin) tokens: Generated by bgutil-pot for BotGuard bypass
  - Session cookies: Extracted by yt-dlp, passed to FFmpeg for format 18 fallback
- **Key Files:**
  - `audio_player.py`:
    - `get_audio_url()` - Metadata and stream URL extraction (line 35-175)
    - `_start_ytdlp_stream()` - yt-dlp subprocess piping (line 178-226)
    - `extract_playlist_info()` - Playlist metadata extraction
  - `main.py` - bgutil-pot HTTP server startup (line 18-36)
- **Features:**
  - YouTube search: `ytsearch:query`
  - Direct URL support: YouTube, music.youtube.com
  - Playlist extraction: `list=...` parameter support
  - Format selection: Best audio quality + fallback to video+audio (format 18)
  - HTTP headers: User-Agent, Cookie, Referer, Origin (whitelisted)
  - Thumbnail extraction: Cached on Track object
- **Authentication Chain:**
  1. bgutil-pot generates PO token → URL contains `pot=...`
  2. yt-dlp extracts session cookies from `.yt-dlp/` cache
  3. FFmpeg receives filtered HTTP headers for CDN authentication
- **Rate Limits:**
  - YouTube throttling via IP (handled by yt-dlp automatic retry)
  - No explicit quota enforcement in this bot
- **Data Caching:** yt-dlp cache at `~/.yt-dlp/` (cookies, player JS)

### SoundCloud (via yt-dlp)
- **Purpose:** Audio extraction from SoundCloud URLs and playlist support
- **SDK/Client:** Built-in yt-dlp extractor (requires no separate auth)
- **Auth:** None required
- **Key Files:**
  - `audio_player.py` - SoundCloud detection in `is_playlist_url()` (line 231-239)
- **Features:**
  - Direct URL support: `soundcloud.com/...`
  - Playlist support: `/sets/...` paths
  - Format: Direct audio stream (no signature solving needed)
- **Rate Limits:** SoundCloud's standard rate limiting (transparent via yt-dlp)

### Other Platforms (via yt-dlp)
- **Supported:** Any platform with yt-dlp extractor (Spotify preview, BandCamp, etc.)
- **SDK/Client:** Built-in yt-dlp extractors
- **Auth:** Per-platform (typically none for free streams)
- **Detection:** Auto-handled by yt-dlp's extractor selection

## Data Storage

### Local Filesystem (JSON)
- **File:** `guild_settings.json` (location configurable via `GUILD_SETTINGS_FILE` env var)
- **Purpose:** Persist per-guild configuration
- **Structure:**
  ```json
  {
    "<guild_id>": {
      "allowed_channel": "<channel_id>",
      "bitrate": 128,
      "admins": ["<user_id>", "<user_id>"]
    }
  }
  ```
- **Read/Write Functions:** `guild_settings.py` (lines 10-75)
  - `load_settings()` - Read JSON file
  - `save_settings(data)` - Write JSON file
  - Per-setting accessors: `get_allowed_channel()`, `set_allowed_channel()`, etc.
- **Default Location:** Project root (local dev) or `/data/guild_settings.json` (Docker)
- **Persistence:** Manual JSON write-through (no ORM, no database)
- **Backup:** None (file-based, no replication)

### Audio Stream (Memory)
- **Source:** yt-dlp stdout → FFmpeg stdin pipe
- **Storage:** Streamed to Discord voice socket (no persistent cache)
- **Buffering:** Handled by discord.py's voice client (ring buffer, implementation detail)
- **Duration:** Single play session (stops on skip/stop command)

### Configuration (YAML)
- **File:** `config.yaml`
- **Purpose:** Runtime configuration (bot token, Discord settings, audio params)
- **Parser:** `pyyaml`
- **Persistence:** Manual (no auto-save; requires edit + bot restart)

## Authentication Providers

### Discord OAuth2 (Bot Token)
- **Type:** Bot authentication (not user OAuth)
- **Credential:** `bot_token` in `config.yaml`
- **Scope:** Implicit in bot invite URL (managed externally)
- **Location:** Loaded in `main.py` line 78
- **Env Override:** None (YAML-only)

### YouTube Session Cookies & PO Tokens
- **Type:** Hybrid (session cookies + generated tokens)
- **Cookies:** Extracted by yt-dlp from `~/.yt-dlp/` cache (auto-managed by yt-dlp)
- **PO Tokens:** Generated by bgutil-pot CLI on demand
- **Lifecycle:** Ephemeral per stream request (not cached between plays)
- **Failure Mode:** Degrades to format 18 (video+audio, ~128kbps AAC) if token generation fails

## Background Services

### bgutil-pot HTTP Server
- **Purpose:** Optional PO token generation via HTTP (legacy, often broken)
- **Startup:** `main.py` lines 18-36
- **Command:** `bgutil-pot.exe server --host 127.0.0.1`
- **Port:** 4416 (default, defined in plugin)
- **Status Check:** Plugin health check at `/ping` endpoint
- **Fallback:** If unavailable, CLI provider (preference 1) is used automatically
- **Shutdown:** Graceful termination on bot exit (`_bgutil_proc.terminate()`)

## External Binary Dependencies

### FFmpeg
- **Purpose:** Audio codec transcoding and piping
- **Resolution:** 
  1. `config.yaml` `ffmpeg_path` (if not "ffmpeg")
  2. `shutil.which("ffmpeg")` from PATH
  3. `imageio_ffmpeg.get_ffmpeg_exe()` fallback
- **Key Command Format:**
  ```
  ffmpeg -i "pipe:0" -f s16le -ar 48000 -ac 2 "pipe:1"
  ```
- **Input:** Binary audio stream from yt-dlp subprocess stdout
- **Output:** PCM/s16 stereo at 48kHz to Discord voice socket
- **HTTP Headers:** Receives filtered set from yt-dlp (User-Agent, Cookie, Referer, Origin)

### Deno (JavaScript Runtime)
- **Purpose:** YouTube signature solving (needed by yt-dlp-ejs)
- **Installation:** Docker only (`curl -fsSL https://deno.land/install.sh | sh`)
- **Location:** `/root/.deno/bin` (Docker) or `~/.deno/bin` (local setup)
- **PATH Setup:** `main.py` lines 13-16 ensures `/root/.deno/bin` is in PATH before yt-dlp import
- **Invocation:** Called transparently by yt-dlp when signature solving needed

### bgutil-pot (YouTube PO Token Generator)
- **Purpose:** Generate PO tokens for BotGuard bypass on YouTube
- **Platform Binary:** `bgutil-pot.exe` (Windows, included in repo)
- **Docker Download:** Linux x86_64 binary from GitHub releases
- **Execution Modes:**
  1. Subprocess CLI: Per-request token generation via command-line invocation
  2. HTTP Server: Background daemon on port 4416 (optional, often broken)
- **yt-dlp Integration:** Custom extractor plugin at `yt-dlp-plugins/bgutil-ytdlp-pot-provider/`

## Error Handling & Resilience

**YouTube Stream Failures:**
- HTTP 403 (AVERROR_HTTP_FORBIDDEN): Triggers FFmpeg exit code validation
- PO token absent: Falls back to format 18 (video+audio)
- bgutil unavailable: User warned, continues with weak client fallback
- yt-dlp extraction failure: Exception propagates to command handler

**Voice Connectivity:**
- Voice client disconnection: Detected by `on_voice_state_update` event
- Empty channel timeout: 60-second grace period, then auto-disconnect
- Connection errors: Logged, command response includes error message

**Configuration Errors:**
- Missing `config.yaml`: RuntimeError on bot startup
- Invalid YAML: Caught by `yaml.safe_load()`, error logged
- Missing bot token: RuntimeError on bot startup
